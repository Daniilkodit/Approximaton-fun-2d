# Approximation-fun-2d

Многопоточная программа на C++ для аппроксимации функций двух переменных методом конечных элементов (МКЭ) на треугольной сетке.

Область определения — прямоугольник `[a, b] × [c, d]`, разбиваемый на `nx × ny` прямоугольных ячеек, каждая из которых делится на треугольники. Система линейных уравнений хранится в формате MSR (Modified Sparse Row) и решается итерационным методом минимальных невязок с предобуславливанием симметричным методом Зейделя. Вычисления распараллелены с помощью `pthread`.

## Метод

Аппроксимация функции `f(x, y)` строится как линейная комбинация кусочно-линейных базисных функций на треугольной сетке. Алгоритм включает следующие этапы:

1. Построение разреженной матрицы Грама в формате MSR (заполнение диагонали и внедиагональных элементов по вкладам от каждого треугольника).
2. Формирование правой части — интегрирование произведения `f(x,y)` на базисные функции по каждому треугольному элементу.
3. Решение СЛАУ итерационным методом минимальных невязок с предобуславливателем (симметричный Зейдель).
4. Вычисление невязки аппроксимации в различных нормах.

## Тестовые функции

Набор тестовых функций задаётся параметром `func_id` (k = 0, 1, 2, …). Как минимум реализованы полиномиальные функции для верификации точности метода.

## Многопоточность

Программа использует `pthread` для распараллеливания:

- Заполнение матрицы и правой части — каждый поток обрабатывает свой диапазон строк.
- Матрично-векторные операции (умножение, скалярное произведение, нормы) — разделение по строкам между потоками.
- Синхронизация через `reduce_sum` (барьерная редукция для суммирования частичных результатов).

## Проверки корректности

Модуль `check_alg` содержит автоматические проверки построенной матрицы: контроль суммы строк и проверка симметрии. Эти проверки позволяют убедиться в корректности заполнения матрицы до запуска решателя.

## Зависимости

- Компилятор C++ с поддержкой C++11 или выше (g++)
- POSIX Threads (`pthread`)
- Linux (используются `sys/sysinfo.h`, `sys/resource.h`, `sched.h`)

## Сборка и запуск

```bash
make
./a.out a b c d nx ny func_id eps maxit p
```

| Параметр  | Описание                                      |
|-----------|-----------------------------------------------|
| `a`, `b`  | Границы области по x: [a, b]                  |
| `c`, `d`  | Границы области по y: [c, d]                  |
| `nx`      | Число разбиений по x                          |
| `ny`      | Число разбиений по y                          |
| `func_id` | Номер тестовой функции                        |
| `eps`     | Точность итерационного метода                 |
| `maxit`   | Максимальное число итераций                   |
| `p`       | Количество потоков                            |

Пример:

```bash
./a.out 0 1 0 1 10 10 0 1e-10 1000 4
```

Аппроксимация на единичном квадрате с сеткой 10×10, функция k=0, точность 10⁻¹⁰, до 1000 итераций, 4 потока.

Дополнительные цели сборки:

```bash
make clean     # очистка объектных файлов
make rebuild   # полная пересборка
make debug     # сборка с флагом -DDEBUG
```

## Структура проекта

```
├── header.h                                       — общие заголовки и объявления
├── makefile                                       — сборка проекта
├── main/
│   ├── main.cpp                                   — точка входа, разбор аргументов
│   ├── thread.cpp                                 — функция потока (thread_fun)
│   ├── time.cpp                                   — замер времени (wall-clock и CPU)
│   └── reduce_sum.cpp                             — барьерная редукция для потоков
├── fill_msr_matrix/
│   ├── fill_msr_matrix.cpp                        — заполнение MSR-матрицы
│   ├── calculation_сourrant_basis.cpp             — вычисление на базисных функциях
│   └── fill_right_side.cpp                        — формирование правой части
├── residual/
│   ├── functions.cpp                              — тестовые функции f(x, y)
│   ├── calculate_approximation_fun.cpp            — вычисление аппроксимации в точке
│   └── claculate_residual.cpp                     — вычисление невязки
├── check_alg/
│   └── check_symm_and_rov_sum.cpp                 — проверка симметрии и сумм строк
└── solve_msr_system/
    ├── minimal_errors_method.cpp                  — метод минимальных невязок
    ├── operations_matrix_or_vec.cpp               — матрично-векторные операции
    └── symm_seidel_preconditioner.cpp             — симметричный предобуславливатель Зейделя
```

## Вывод программы

Программа выводит в консоль:

- Количество итераций решателя
- Время работы (wall-clock и CPU)
- Невязку аппроксимации в нескольких нормах (r1, r2, r3, r4)
